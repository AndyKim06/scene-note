## 과제 개요

> **맞습니다.** 중간과제는 **로그인 페이지** + **로그인 성공 후 보여줄 메인 페이지**를 제작하고,  
> GitHub에 올린 뒤 Vercel로 배포하는 것입니다.  
> 강의 슬라이드(p.27)에서 예시로 Netflix 스타일 메인 화면을 보여주고 있습니다.

## 선택한 메인 페이지 아이디어

### 🎬 SceneNote — 단편영화 기획 노트 툴

로그인하면 단편영화 기획을 정리할 수 있는 대시보드가 나옵니다.  
데이터는 `localStorage`에 저장되므로 새로고침해도 유지됩니다.

**4개 탭 섹션으로 구성:**

| 탭 | 내용 |
|---|---|
| 스토리보드 | 씬 카드 추가/삭제. 씬번호(자동)·장소·등장인물·내용·이미지URL·예상 시간(분) 입력 |
| 아이디어 노트 | 포스트잇 스타일 자유 메모 추가/삭제 |
| 회의록 | 날짜·제목·내용으로 회의 기록 추가/삭제 |
| 소스 라이브러리 | 이미지/영상/음악 URL + 설명 + 유형 태그로 참고 소스 저장 |

**구현 난이도:** ★★★☆☆

## 필요한 파일 구조

```
myFirstApp/
├── index.html        ← 로그인 페이지 (첫 화면)
├── main.html         ← SceneNote 메인 대시보드
├── style.css         ← 공통 스타일
├── script.js         ← 로그인/로그아웃 + 앱 전체 로직
└── package.json      ← Node.js 프로젝트 파일
```

```

요구사항:

- 배경에 그라디언트 또는 필름 느낌의 서브 컬러 포인트 추가

- 상단에 "SceneNote" 서비스명 (흰색, 굵게)
- 부제목: "단편영화 기획을 한 곳에" (작은 글씨, 회색)

- 버튼 hover 시 살짝 밝아지는 효과
- 하단에 작은 안내 텍스트: "테스트 계정: test@test.com / 1234"
- 반응형 디자인 (모바일에서도 잘 보이게)
- script.js 파일 연결: <script src="script.js"></script>
```

### STEP 2 — 로그인 로직 만들기 (`script.js`)

**해야 할 일:** 로그인 검증, localStorage 저장/불러오기, 페이지 이동, 로그아웃, 4개 섹션 데이터 로직 전부 구현

**Claude 프롬프트:**

```
SceneNote 앱의 전체 로직을 담는 script.js 파일을 만들어줘.

--- [로그인/로그아웃] ---
- 하드코딩 계정: 이메일 "test@test.com", 비밀번호 "1234"
- 빈 칸 제출 시: alert("이메일과 비밀번호를 입력해주세요")
- 틀린 경우: alert("이메일 또는 비밀번호가 올바르지 않습니다")
- 로그인 성공 시: localStorage에 isLoggedIn = "true" 저장 후 main.html로 이동
- main.html 로드 시: isLoggedIn 확인, 없으면 index.html로 리다이렉트
- logout() 함수: localStorage.clear() 후 index.html로 이동

--- [탭 전환] ---
- showTab(tabName) 함수: 탭 클릭 시 해당 섹션만 보이고 나머지는 숨김
- 기본 활성 탭: "storyboard"

--- [스토리보드] ---
localStorage 키: "sn_scenes" (JSON 배열)
각 씬 객체: { id, sceneNumber, location, characters, description, imageUrl, duration }
- addScene(): 새 씬 객체 생성 후 저장, renderScenes() 호출
- deleteScene(id): 해당 id 삭제 후 renderScenes() 호출
- saveScene(id, field, value): 특정 필드 값 변경 시 저장 (oninput으로 호출)
- renderScenes(): scenes-container에 씬 카드 HTML 동적 생성
  카드 HTML 구조:
  - "SCENE 01" 형식 씬 번호 제목
  - 장소 input (oninput="saveScene(id,'location',this.value)")
  - 등장인물 input (oninput="saveScene(id,'characters',this.value)")
  - 내용 textarea (oninput="saveScene(id,'description',this.value)")
  - 이미지 URL input (oninput="saveScene(id,'imageUrl',this.value)")
  - 이미지 URL 입력 시 카드 내 img 태그 src 업데이트 (미리보기)
  - 예상 시간 number input (단위: 분, min=0, oninput="saveScene(id,'duration',this.value)") + "분" 텍스트
  - "씬 삭제" 버튼 (onclick="deleteScene(id)")

--- [아이디어 노트] ---
localStorage 키: "sn_memos" (JSON 배열)
각 메모 객체: { id, content, createdAt }
- addMemo(): 입력창(id="memo-input")의 값으로 메모 추가, 빈 값이면 무시
- deleteMemo(id): 삭제
- renderMemos(): memos-container에 메모 카드 HTML 생성
  카드: 내용 텍스트, 작성일시(예: 2025.04.21 14:30), "삭제" 버튼

--- [회의록] ---
localStorage 키: "sn_meetings" (JSON 배열)
각 회의록 객체: { id, date, title, content }
- toggleMeetingForm(): 회의록 입력 폼 show/hide
- addMeeting(): 날짜·제목·내용 입력값으로 회의록 추가, 폼 초기화
- deleteMeeting(id): 삭제
- renderMeetings(): meetings-container에 회의록 카드 HTML 생성
  카드: 날짜 + 제목(굵게) + 내용 + "삭제" 버튼

--- [소스 라이브러리] ---
localStorage 키: "sn_sources" (JSON 배열)
각 소스 객체: { id, url, description, type }
type 종류: "이미지" | "영상" | "음악" | "기타"
- toggleSourceForm(): 소스 입력 폼 show/hide
- addSource(): URL·설명·유형 입력값으로 소스 추가, 폼 초기화
- deleteSource(id): 삭제
- renderSources(): sources-container에 소스 카드 HTML 생성
  카드: URL 링크(클릭 시 새 탭 오픈), 설명, 유형 뱃지, "삭제" 버튼

페이지 로드 시 자동으로 renderScenes(), renderMemos(), renderMeetings(), renderSources() 실행해줘.
```

**해야 할 일:** SceneNote 대시보드 UI 구현 (4탭 구조)

**Claude 프롬프트:**

```
SceneNote 메인 대시보드 main.html을 만들어줘.
script.js의 함수들이 이미 완성돼 있으니 UI와 id, onclick만 정확히 맞춰줘.

- 배경색: #1a1a2e
- 텍스트: 흰색 계열

--- [네비게이션바] ---
- 왼쪽: SceneNote (흰색 굵은 텍스트)
- 오른쪽: "로그아웃" 버튼 (onclick="logout()")

--- [탭 메뉴] ---
탭 4개:  스토리보드 /  아이디어 노트 /  회의록 /  소스 라이브러리
- 클릭 시 onclick="showTab('storyboard')" 형식으로 연결
- 활성 탭은 포인트 컬러 하이라이트
- 기본 활성: 스토리보드

--- [스토리보드 섹션 (id="tab-storyboard")] ---
- 상단: "+ 씬 추가" 버튼 (onclick="addScene()")
- 씬 카드 컨테이너: id="scenes-container"
- 카드 레이아웃: 2열 그리드 (모바일에서는 1열)
- 각 카드 스타일: 어두운 카드 배경(#16213e), 둥근 모서리, 패딩
- 이미지 미리보기 영역: 카드 상단에 img 태그 (이미지 없으면 회색 빈 영역)

--- [아이디어 노트 섹션 (id="tab-memos", style="display:none")] ---
- 상단: textarea (id="memo-input", placeholder="자유롭게 아이디어를 적어보세요...") + "+ 추가" 버튼 (onclick="addMemo()")
- 메모 컨테이너: id="memos-container"
- 메모 카드: 포스트잇 느낌 (#2a2a1a 배경), 왼쪽 포인트 컬러 border

--- [회의록 섹션 (id="tab-meetings", style="display:none")] ---
- 상단: "+ 회의록 추가" 버튼 (onclick="toggleMeetingForm()")
- 입력 폼 (id="meeting-form", 기본 숨김):
  - date input (id="meeting-date")
  - text input (id="meeting-title", placeholder="회의 제목")
  - textarea (id="meeting-content", placeholder="회의 내용")
  - "저장" 버튼 (onclick="addMeeting()")
- 회의록 컨테이너: id="meetings-container"
- 회의록 카드: 날짜(작은 회색) + 제목(굵게) + 내용 + 삭제 버튼

--- [소스 라이브러리 섹션 (id="tab-sources", style="display:none")] ---
- 상단: "+ 소스 추가" 버튼 (onclick="toggleSourceForm()")
- 입력 폼 (id="source-form", 기본 숨김):
  - text input (id="source-url", placeholder="URL을 입력하세요")
  - text input (id="source-desc", placeholder="설명")
  - select (id="source-type"): 옵션 이미지/영상/음악/기타
  - "저장" 버튼 (onclick="addSource()")
- 소스 컨테이너: id="sources-container"
- 소스 카드: 3열 그리드, URL 링크 + 설명 + 유형 뱃지(색상 구분) + 삭제 버튼

모든 id와 onclick 함수명은 위에 적힌 것과 정확히 일치시켜줘.
```

### STEP 5 — GitHub Repository 만들고 코드 올리기

**해야 할 일:** GitHub에 저장소 만들고 프롬프트로 연결 & push

**방법:**
1. github.com 접속 → "New repository" 클릭
2. Repository name: `scene-note` 입력
3. Public 선택 → "Create repository" 클릭
4. 화면에 나오는 커맨드를 VS Code 프롬프트창에 붙여넣기 → Accept 클릭

**Claude 프롬프트 (VS Code 프롬프트창 실행용):**

```
현재 폴더의 파일들을 GitHub에 올려줘.
다음 명령어들을 순서대로 실행해줘:

1. git init
2. git add .
3. git commit -m "feat: SceneNote 로그인 및 기획 대시보드 구현"
4. git branch -M main
5. git remote add origin https://github.com/[내_깃헙_아이디]/scene-note.git
6. git push -u origin main

[내_깃헙_아이디] 부분을 내 실제 GitHub 아이디로 바꿔서 실행해줘.
```

### STEP 6 — Vercel로 배포하기

**해야 할 일:** Vercel에서 GitHub 연동 후 자동 배포

**방법:**
1. vercel.com 접속 → "Sign Up" → "Continue with GitHub"
2. "Authorize Vercel" 클릭
3. "Import Git Repository" → `scene-note` 선택 → "Import"
4. "Deploy" 버튼 클릭
5. 배포 완료 후 생성된 URL (예: `scene-note.vercel.app`) 접속 확인

> 이후 코드를 수정하고 GitHub에 push하면 Vercel이 자동으로 재배포합니다.

## 난이도 걱정될 때 단순화 방법

막히면 아래 순서로 기능을 줄여가면 됩니다.

| 단계 | 제거할 것 | 결과 |
|---|---|---|
| 1단계 | 소스 라이브러리 섹션 제거 | 3개 탭만 |
| 2단계 | 회의록 섹션도 제거 | 스토리보드 + 아이디어 노트만 |
| 3단계 | 씬 카드 자동저장 → 저장 버튼으로 변경 | oninput 제거, 버튼 추가 |
| 4단계 | localStorage 전체 제거 | 새로고침 시 초기화되어도 OK |

**단순화가 필요할 때 Claude 프롬프트:**

```
main.html과 script.js에서 소스 라이브러리 섹션과 회의록 섹션을 완전히 제거해줘.
스토리보드와 아이디어 노트 2개 탭만 남기고, 나머지 기능은 그대로 유지해줘.
```

## 주의사항

- Vercel 배포 URL을 과제 제출 시 함께 제출하세요.
- 로그인 없이 `main.html`에 직접 접근하면 자동으로 `index.html`로 이동해야 합니다.
- `localStorage`는 같은 브라우저에서만 유지됩니다. 다른 기기에서 열면 데이터가 초기 상태입니다.
- 이미지/영상 소스는 URL 입력 방식이므로 파일 직접 업로드는 지원하지 않습니다.
